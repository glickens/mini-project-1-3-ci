# Automated CI Pipeline (GitHub Actions - Python)

## Description
This project uses GitHub Actions to implement a Continuous Integration (CI) pipeline that automatically installs dependencies, runs tests, and builds the application on every push or pull request. It ensures code is validated before merging, improving reliability and reducing integration issues.
## CI Workflow
- Trigger: Push and Pull Requests to main
- Install dependencies from requirements.txt
- Run automated tests using pytest
